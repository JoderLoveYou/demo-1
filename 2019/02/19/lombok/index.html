lombok说明

Lombok使用

Lombok是一款好用顺手的工具。Lombok是一种Java实用工具，可用来帮助开发人员消除Java的冗长代码，尤其是对于简单的Java对象（POJO）。它通过注释实现这一目的。通过在开发环境中实现Lombok，开发人员可以节省构建诸如hashCode()和equals()这样的方法减少大量时间和让代码看起来更加简洁。
其实现原理利用javac编译器对源码进行分析，生成一个抽象的语法树（AST)，通过Pluggable Annotation Processing API来实现代码生成的。

准备

  • lombok
  • idea

idea需要安装插件，代码才能正常显示

这里只是用了maven引入相关的jar包，

@Getter/@Setter

使用@Getter可以生成model的get方法，使用@Setter可以生成model的set方法；用在属性上，再也不用自己手写setter和getter方法了，还可以指定访问范围

相当于

@ToString

用在类上，可以自动覆写toString方法，当然还可以加其他参数，例如@ToString(exclude=”id”)排除id属性，或者@ToString(callSuper=true, includeFieldNames=true)调用父类的toString方法，包含所有属性

结果

@EqualsAndHashCode

用在类上，自动生成equals方法和hashCode方法

@Data

其实以上的注解，可以用Data注解代替。注解在类上，相当于同时使用了@ToString、@EqualsAndHashCode、@Getter、@Setter和@RequiredArgsConstrutor这些注解，对于POJO类十分有用

@NoArgsConstructor, @RequiredArgsConstructor and @AllArgsConstructor

用在类上，自动生成无参构造和使用所有参数的构造函数以及把所有@NonNull属性作为参数的构造函数，如果指定staticName = “of”参数，同时还会生成一个返回类对象的静态工厂方法，比使用构造函数方便很多

@Value

用在类上，是@Data的不可变形式，相当于为属性添加final声明，只提供getter方法，而不提供setter方法

@Builder

用在类、构造器、方法上，为你提供复杂的builder APIs

相当于

@Cleanup

自动管理资源，用在局部变量之前，在当前变量范围内即将执行完毕退出之前会自动清理资源，自动生成try-finally这样的代码来关闭流

相当于

@SneakyThrows

自动抛受检异常，而无需显式在方法上使用throws语句

相当于

@Synchronized

用在方法上，将方法声明为同步的，并自动加锁，而锁对象是一个私有的属性$lock或$LOCK，而java中的synchronized关键字锁对象是this，锁在this或者自己的类对象上存在副作用，就是你不能阻止非受控代码去锁this或者类对象，这可能会导致竞争条件或者其它线程错误

相当于

@Log

根据不同的注解生成不同类型的log对象，但是实例名称都是log，
有六种可选实现类

val/var

val用在局部变量前面，相当于将变量声明为final;var用在局部变量前面
